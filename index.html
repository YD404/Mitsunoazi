Project "Mitsunoazi"

ビジュアル仕様書

1. プロジェクト概要

複数のウェブカメラから入力された映像をキャプチャし、ユーザーが選択したStatus（Crazy / Attacker / Blocker / Healer）を付与します。確定された画像は、対応するタイムライン演出で再生された後、サブディスプレイに累積表示されます。

2. システム要件

Unityバージョン

2022.3.6f1 (2D)

ディスプレイ解像度

1920 x 1080 (想定)

Webカメラ解像度

640 x 360

3. 主要フロー

ユーザーが数字キー (0–9)を押すと、対応するカメラで映像キャプチャを開始し、状態が Processing へ移行します。

Unity内で輝度に基づき、画素の透過処理を行います。

生画像を ImageCapture/ に、透過後画像を ImageStaged/ に保存します。

Processing 中、または処理完了後の SelectingStatus 状態中に、数字キーで Status を選択します。

A–Eキーで画像を確定します。Processing 中に押された場合は「確定予約」として保持し、処理完了時に自動実行されます。

ImageStaged/ 内のファイル名にステータス名を付与し、ImageConfirmed/ へ移動します。

確定したステータスに応じたプレハブからランダムに1つを選択し、演出を再生します。

タイムライン再生終了後、該当画像をサブディスプレイへ追加表示します。

4. 機能仕様詳細

4.1. 入力（キーマップ）

割り当て キャプチャ / ステータス選択 確定キー 備考
カメラ 00, 1A0: 正順, 1: 逆順
カメラ 12, 3B2: 正順, 3: 逆順
カメラ 24, 5C4: 正順, 5: 逆順
カメラ 36, 7D6: 正順, 7: 逆順
カメラ 48, 9E8: 正順, 9: 逆順

ステータス選択順序

正順: Crazy → Attacker → Blocker → Healer → ...

逆順: Crazy → Healer → Blocker → Attacker → ...

4.2. 状態管理 (カメラ単位)

Ready

数字キー押下で `Processing` へ

Processing

画像処理完了後、予約がなければ `SelectingStatus` へ、予約があれば `Ready` へ

SelectingStatus

確定キーで確定処理後、 `Ready` へ

4.3. 画像処理・ファイル管理

輝度 (0.299×R + 0.587×G + 0.114×B) が上位10%以上の画素を透明化します。

ImageCapture/: キャプチャした生のPNG画像

ImageStaged/: 透過処理後の一次保存PNG画像 (webcam_<camIdx>_<timestamp>.png)

ImageConfirmed/: ステータス確定後の最終保存PNG画像 (webcam_<camIdx>_<timestamp>_<Status>.png)

4.4. タイムライン再生

`PlayableDirector`で再生。`TimelineInfo`が参照する`SpriteRenderer`に画像を差し替え。再生停止時にサブディスプレイへ通知後、インスタンスを破棄します。

4.5. サブディスプレイ表示

起動時に`ImageConfirmed/`内の全画像を表示。タイムライン停止時に`AddNewImage(path)`を介して画像を追加します。

5. スクリプト構成と役割

5.1. フォルダ構成 (主要部分)

Assets/
├── Scripts/
│   ├── Add/
│   ├── Core/
│   ├── DevOnly/  # 現状スクリプトなし
│   └── Sys/
├── StreamingAssets/
│   ├── ImageCapture/
│   ├── ImageConfirmed/
│   └── ImageStaged/
└── ... (Prefabs, Scenes, etc.)

5.2 Core スクリプト

  • WebcamCaptureManager: WebCamTextureでキャプチャ実行、生PNG保存。
  • CaptureStateManager: カメラ状態と確定予約を管理。ファイル移動とTimelinePlayer呼出。
  • ImageProcessor: 画像の透過処理とPNG保存、ファイル移動を担当。
  • StatusManager: Status列挙型の管理と循環選択APIを提供。
  • TimelinePlayer: プレハブ再生、Sprite差替、インスタンス破棄を管理。
  • TimelineInfo: SpriteRendererへの参照を保持。

5.3 Sys スクリプト

  • CleanupAssetsOnQuit: 終了時に一時保存フォルダをクリーンアップ。
  • SubDisplayManager: サブディスプレイ有効化と画像表示・管理。
  • BgmLooper: BGMループ再生と音量調整機能を提供。

5.4 Add スクリプト

  • SubImagePosition: サブディスプレイ上の画像の初期座標を記録。
  • SubImageMotion: Perlinノイズで画像を微小に動かす。
  • SubImageStart: 画像登場時のアニメーションを制御。

7. Git管理

7.1. Gitからのクローン方法

  1. Gitをインストールしたのち、Unityのプロジェクトを保存したい場所でターミナルを開きます。
  2. 以下のコマンドを実行して、リポジトリをローカルにクローンします。
    git clone https://github.com/YD404/Mitsunoazi.git
  3. UnityHubを開き、「開く」→「ディスクから加える」で、ローカルにクローンしたプロジェクトフォルダを選択します。
  4. Googleドライブからzipファイルをダウンロードします。
  5. Unityでプロジェクトが開いたら、ダウンロードしたzipファイルの中身を、プロジェクト内の同じ名前の場所にコピー＆ペーストしてください。
